# 완장봇 1.3.1 로드맵

## 목표
1.3.0에서 이월된 기능과 새로 추가된 요구사항을 무리하게 한 번에 섞지 않고, 사용자 체감이 큰 기능부터 단계적으로 반영한다.

이 로드맵은 1.3.1 작업 중 계속 갱신하는 기준 문서로 사용한다.

---

## 핵심 원칙
- 기존 1.3.0에서 정리한 공통 차단/삭제 정책 구조를 재사용한다.
- UI는 처음 사용하는 사람도 바로 이해할 수 있게 유지한다.
- 새 기능은 가능한 한 설정 / 저장 / 실행 / 로그 흐름이 한 번에 이어지도록 구현한다.
- beta workflow(빌드 → 업로드 → BETA_NOTES 기록 → 커밋)를 계속 유지한다.

---

## 1.3.1 주요 작업 축

### A. 목록 감시 범위 확장
목표
- 일반 목록뿐 아니라 개념글 페이지, 특정 말머리 페이지도 감시 대상으로 지원한다.

세부 방향
- URL 규칙 파악
- 목록 타입 구분값 추가
  - 일반 목록
  - 개념글 목록
  - 특정 말머리 목록
- UI에서 사용자가 어떤 목록을 감시하는지 분명히 알 수 있게 구성
- 중복 감지 방지 정책 정리

우선순위
- 높음

---

### B. 도배 방지 1차
목표
- 짧은 시간 안에 유동/깡계 게시글이 급증하는 상황을 도배로 감지하고, 그 시점 이후 글부터 강화 규칙을 적용한다.

핵심 정책
- 탐지 우선 대상은 유동 / 깡계
- 강화 조치도 유동 / 깡계 중심
- 도배 시작 이전 글에는 소급 적용 금지
- 도배 시작 이후 감지된 글부터만 강화 규칙 적용

예상 설정 항목
- 도배 방지 사용 여부
- 감지 기준 시간(분)
- 유동 글 감지 임계치
- 깡계 글 감지 임계치
- 도배 대응 방식
  - 삭제 전용
  - 차단 강화
  - AI 필터 병행
- 적용 대상
  - 유동만
  - 깡계만
  - 유동 + 깡계

우선순위
- 높음

---

### C. 해외 IP 차단
목표
- 작성자 IP가 국내 대역이 아닌 경우 차단할 수 있도록 한다.

세부 방향
- 작성자 IP 추출 경로 재확인
- 국내/해외 판별 로직 추가
- 설정 UI 추가
- 활동 로그/차단 이력 문구 정리

주의
- 1차는 IPv4 기준
- 불명확한 형식은 보수적으로 처리

우선순위
- 중간

---

### D. 갤러리 설정 자동 갱신
목표
- 갤러리 설정 변화가 생겨도 수동 재입력 부담을 줄인다.

세부 방향
- 설정 확인 요청 경로 정리
- 자동 갱신 주기/조건 정의
- 실패 시 재시도 및 로그 정리
- UI 노출 범위 검토

우선순위
- 중간

---

### E. 금지어 필터 특수 조건 보강
목표
- 금지어 필터를 더 정교하게 운영할 수 있도록 남은 특수 조건을 보강한다.

세부 방향
- 현재 구현 범위 재점검
- 남은 조건 항목 확정
- UI / 저장 / 실행 경로 연결
- 활동 로그와 차단 사유 문구 정리

우선순위
- 중간

---

## 권장 구현 순서
### Phase 1. 준비
- 1.3.1 beta 라인 전환
- 로드맵 확정
- 범위별 코드 영향 지점 확인

### Phase 2. 목록 감시 범위 확장
- 개념글 / 특정 말머리 페이지 URL 규칙 파악
- 목록 타입 처리 구조 정리
- UI와 저장 구조 연결

### Phase 3. 도배 방지 1차
- 상태값 설계
- 유동 / 깡계 급증 탐지
- 도배 시작 이후 글만 강화 적용
- 활동 로그 정리

### Phase 4. 해외 IP 차단
- 판별 로직 / 설정 / 실행 연결

### Phase 5. 갤러리 설정 자동 갱신
- 요청 / 주기 / 실패 처리 정리

### Phase 6. 금지어 특수 조건 보강
- 남은 조건 연결 및 마무리

### Phase 7. 통합 점검
- beta 빌드
- 업로드
- BETA_NOTES 기록
- 커밋
