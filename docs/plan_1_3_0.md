# 완장봇 1.3.0 작업 계획서

## 기준
- 현재 안정판 기준: 1.2.2
- 이번 1.3.0은 단순 기능 추가보다 차단 집행 구조/필터 설정 구조를 일반화하는 버전으로 본다.
- 우선순위는 `공통 기반 -> 개별 기능` 순서로 진행한다.

---

## 1.3.0 목표 기능
1. 해외 IP 차단 기능
2. 갤러리 설정 자동 갱신 기능
3. 금지어 필터 특수 조건 기능
4. 차단 없이 삭제만 하는 기능
5. 필터별 개별 차단 설정 기능

---

## 실제 코드 기준 핵심 영향 지점

### 1) BotService.kt
예상 영향 범위
- `BotConfig` 확장
- 게시글/댓글 분석 결과 구조 확장
- 실제 집행 함수(`handleBadPost`, `handleBadComment`, `executeBlockRequest` 계열) 분기 확장
- 글/댓글 삭제 전용 요청 경로 추가
- 해외 IP 판정 호출 지점 추가
- 금지어 특수 조건 판정 호출 지점 추가
- 필터별 개별 차단 설정 override 적용 지점 추가
- 갤러리 설정 자동 갱신 스케줄/실행 지점 추가

### 2) BotDetailScreen.kt
예상 영향 범위
- 차단 기본 설정 UI 확장
- 삭제만 실행 토글 추가
- 금지어 필터 특수 조건 UI 추가
- ID/IP 필터 하위에 해외 IP 차단 토글 추가
- 각 필터 하단 개별 차단 설정 UI 추가
- 갤러리 설정 자동 갱신 UI 추가

### 3) SharedPreferences(bot_prefs_$botId)
예상 영향 범위
- 기본 차단 설정 신규 키
- 필터별 개별 차단 설정 키
- 금지어 특수 조건 키
- 해외 IP 차단 키
- 갤러리 설정 자동 갱신 키

## 권장 개발 순서

### Phase 1. 공통 집행 정책 모델 정리
목표
- 기존 "기본 차단 설정 1개" 중심 구조를 정책 객체 기반으로 일반화

구현 항목
- `BotConfig` 안의 기본 차단 설정을 재해석할 수 있는 공통 정책 구조 도입
- 개별 필터 override 여부 + override 설정값 저장 구조 추가
- 집행 경로에서 "기본 설정 vs 필터 override"를 선택하는 공통 함수 추가

산출물
- `resolveActionConfig(...)` 같은 공통 결정 함수
- 차단/삭제/삭제만을 표현할 수 있는 공통 정책 구조

### Phase 2. 삭제만 기능
목표
- 차단 기본 설정에서 삭제만 실행 가능

구현 항목
- 기본 차단 설정에 `deleteOnly` 토글 추가
- 글 삭제 요청 / 댓글 삭제 요청 경로 조사 및 구현
- 로그/알림/이력에서 `차단`과 `삭제만` 구분

주의
- 이미 삭제됨 / 권한 없음 / 세션 만료 응답 로그 강화 필요

### Phase 3. 필터별 개별 차단 설정
목표
- 각 필터별로 기본 설정 대신 개별 설정 사용 가능

구현 항목
- 필터별 `use_custom_action_config` 토글
- 차단 시간 / 삭제 여부 / 삭제만 여부 / 사유 / 알림 정책 등 개별 저장
- 금지어 필터부터 먼저 연결 후 나머지 필터로 확대

### Phase 4. 금지어 필터 특수 조건
목표
- 금지어 필터 발동 조건에 세부 gating 추가

구현 항목
- 마스터 토글
- 유동만 적용
- 깡계만 적용
- 이모지 감지
- 특수문자 감지
- 한자 감지

주의
- 유동/깡계 판정 기준 코드상 실제 값 확인 필요
- 정규식 오탐 최소화 필요

### Phase 5. 해외 IP 차단
목표
- ID/IP 필터 하위 옵션으로 해외 IP 차단 추가

구현 항목
- 작성자 IP 추출 지점 확인
- 한국 IPv4 대역 데이터 파일 포맷 결정
- 앱 내 내장 파일 기반 오프라인 판정 1차 구현
- 한국 배정 대역 밖이면 해외 IP로 취급

주의
- 1.3.0 1차는 IPv4 우선
- 자동 최신화는 후순위

### Phase 6. 갤러리 설정 자동 갱신
목표
- DC 관리자 기능 자동 갱신

구현 항목
- VPN 제한 갱신 요청 분석
- 통신사 IP 제한 갱신 요청 분석
- 이미지/동영상 첨부 제한 갱신 요청 분석
- 자동 갱신 주기/실패시 로그 설계

주의
- 이 기능은 조사/실험 단계 분리 필수
- 실제 응답/유효시간을 먼저 검증해야 함
